## Issue #406 — ci(scorecard): add a minimal CodeQL/SAST workflow

**Task type:** ci
**Branch:** agent/agy/claude-sonnet-4-6/ci/codeql-406

---

### Start here

```bash
git checkout master

git checkout -b agent/agy/claude-sonnet-4-6/ci/codeql-406
```

---

### Goal

Add a minimal CodeQL workflow for Python and Actions (YAML) scanning, running on
PR and weekly schedule. This improves the OpenSSF Scorecard SAST check.

---

### STEP 1 — Create `.github/workflows/codeql.yml`

Create `.github/workflows/codeql.yml` with this exact content:

```yaml
name: CodeQL

on:
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review]
    paths:
      - "**.py"
      - ".github/workflows/**.yml"
      - ".github/workflows/**.yaml"
  schedule:
    - cron: "30 3 * * 1"   # every Monday at 03:30 UTC
  workflow_dispatch:

permissions:
  contents: read

concurrency:
  group: codeql-${{ github.event.pull_request.number || github.sha }}
  cancel-in-progress: true

jobs:
  analyze:
    name: Analyze (${{ matrix.language }})
    runs-on: ubuntu-latest

    permissions:
      contents: read
      security-events: write
      actions: read

    strategy:
      fail-fast: false
      matrix:
        language: [python, actions]
        # actions = GitHub Actions YAML workflow analysis

    steps:
      - name: Checkout
        uses: actions/checkout@df4cb1c069e1874edd31b4311f1884172cec0e10 # actions/checkout@v6

      - name: Initialize CodeQL
        uses: github/codeql-action/init@411bbbe57033eedfc1a82d68c01345aa96c737d7 # github/codeql-action/init@v4
        with:
          languages: ${{ matrix.language }}
          queries: security-extended

      - name: Perform CodeQL Analysis
        uses: github/codeql-action/analyze@411bbbe57033eedfc1a82d68c01345aa96c737d7 # github/codeql-action/analyze@v4
        with:
          category: "/language:${{ matrix.language }}"
```

---

### STEP 2 — Add to quality-gate.yml

Open `.github/workflows/quality-gate.yml`. Find the `required_checks` list (or
the section listing required check names). Add `CodeQL` to the list of required
checks if there is a pattern for adding checks — but ONLY if the existing
quality-gate polls named checks by pattern. If the quality-gate uses
`gh run list` or similar, no change is needed.

If in doubt, do NOT add CodeQL to quality-gate. Leave a comment in the PR body
explaining your choice.
