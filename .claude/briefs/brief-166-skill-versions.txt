### Background

Each skill `SKILL.md` file has a YAML frontmatter block starting with `---`.
Currently, no skill declares its own version. We want to add `version: "1.0.0"`
to the frontmatter of every SKILL.md so users on different kit versions can
track skill drift.

We also want `scripts/validate.sh` to check that every SKILL.md has a `version:`
field (same pattern as the existing `allowed-tools:` check at line ~345).

---

### STEP 1 — Read the current frontmatter format

Run this to see what the frontmatter looks like:
```bash
head -12 .agents/skills/dotnet/SKILL.md
```

It will look like:
```yaml
---
name: dotnet
description: >
  Use when modifying C#, .NET, ...
paths:
  - "**/*.cs"
allowed-tools:
  - "Bash(dotnet build:*)"
---
```

We want to add `version: "1.0.0"` as the **last** field before the closing `---`.

---

### STEP 2 — Add version to all 36 shared skills

The shared skills live in `.agents/skills/*/SKILL.md` (36 files).
Use this bash loop to insert `version: "1.0.0"` before the closing `---` of each frontmatter.

**IMPORTANT**: Only add it to files that do NOT already have a `version:` line.

```bash
for f in .agents/skills/*/SKILL.md; do
    if ! grep -q '^version:' "$f"; then
        # Insert 'version: "1.0.0"' before the second '---' (end of frontmatter)
        # The frontmatter ends at the SECOND occurrence of '^---'
        python3 - "$f" <<'PYEOF'
import sys, re

path = sys.argv[1]
with open(path, 'r', encoding='utf-8') as fh:
    content = fh.read()

# Find the second '---' line that closes the frontmatter
lines = content.split('\n')
dashes = [i for i, l in enumerate(lines) if l.strip() == '---']
if len(dashes) < 2:
    print(f"SKIP (no frontmatter): {path}", file=sys.stderr)
    sys.exit(0)

close_idx = dashes[1]
lines.insert(close_idx, 'version: "1.0.0"')

with open(path, 'w', encoding='utf-8') as fh:
    fh.write('\n'.join(lines))
print(f"Updated: {path}")
PYEOF
    else
        echo "Already has version: $f"
    fi
done
```

### STEP 5 — Update `scripts/validate.sh` to check for version

Read `scripts/validate.sh` first. Find the block around line 345-370 that checks
`allowed-tools:` in each SKILL.md. It looks approximately like:

```bash
if [[ -d "$TARGET/skills" ]] && compgen -G "$TARGET/skills/*/SKILL.md" > /dev/null; then
    skill_at_missing=false
    for f in "$TARGET"/skills/*/SKILL.md; do
        ...
        if ! grep -qE '^allowed-tools:' "$f"; then
            warn "skill missing allowed-tools: $f"
            skill_at_missing=true
        fi
    done
    $skill_at_missing || ok "all shared skills declare allowed-tools"
```

**Add an analogous check** for `version:` immediately after the `allowed-tools` check
(inside the same `for` loop, or as a second loop right below it).

The new check should look like this (adapt to match the existing style):

```bash
    skill_version_missing=false
    for f in "$TARGET"/skills/*/SKILL.md; do
        [[ -f "$f" ]] || continue
        if ! grep -qE '^version:' "$f"; then
            warn "skill missing version field: $f"
            skill_version_missing=true
        fi
    done
    $skill_version_missing || ok "all shared skills declare version"
```

Also apply the same check for `.agents/skills/*/SKILL.md` if the validate script
has a separate block for that path (search for `.agents/skills`). Read carefully —
the validate script may check multiple skill paths.
