### Background

Skills with long SKILL.md files load their entire content every time they match
a file path. 22 skills currently exceed 200 lines. Heavy skill loading inflates
context cost. The fix: keep SKILL.md ≤ 80 lines (fast, always loaded) and move
detailed how-to content into a SKILL.deep.md file (opt-in, only read when needed).

---

### Which skills to split

Run this to confirm the list:
```bash
wc -l .agents/skills/*/SKILL.md | sort -rn | awk '$1 > 200 && $2 != "total"'
```

You should see (approximately) 22 skills over 200 lines:
ai-dev, angular, java-kotlin, vue, graphql, react, testing, architecture,
security, svelte, monorepo, performance, infrastructure, accessibility,
api-design, database, observability, node, code-review, error-handling,
i18n, messaging

**Scope of this PR**: split ALL of them (same mechanical pattern for each).

---

### SKILL.md structure after split (target: ≤ 80 lines)

Keep in SKILL.md:
1. YAML frontmatter (unchanged — do NOT touch name/description/paths/allowed-tools/version)
2. `# <Skill Name>` heading
3. `## Goal` — 2–4 sentence summary of what the skill does (keep existing text, trim to essentials)
4. `## Quick reference` — a table or short bullet list of the 5–8 most common commands/patterns
5. `## Full guidance` — one-liner pointing to SKILL.deep.md:
   ```markdown
   ## Full guidance
   Extended how-to, patterns, anti-patterns, and checklists: [`SKILL.deep.md`](SKILL.deep.md)
   ```

Everything else (detailed sections, long checklists, code examples, anti-patterns,
extended explanations) goes into SKILL.deep.md.

---

### SKILL.deep.md structure

```markdown
# <Skill Name> — Deep Reference

> Loaded on demand. The slim [`SKILL.md`](SKILL.md) covers the quick reference.

[all the detailed sections from the original SKILL.md, starting after ## Goal]
```

The deep file does NOT need YAML frontmatter — it's not auto-loaded by Claude Code.

---

### Where to create the files

For each skill in `.agents/skills/<name>/`:
- Edit `.agents/skills/<name>/SKILL.md` in-place (slim version)
- Create `.agents/skills/<name>/SKILL.deep.md` (detailed version)

Do NOT touch `.agy/skills/`, `.claude/skills/`, or `skills/` copies —
those are installed copies synced by install.sh, not the sources. Only the
`.agents/skills/` source files need the split.

---

### STEP-BY-STEP for each skill

For each skill (repeat for all 22):

1. Read the current `.agents/skills/<name>/SKILL.md`
2. Identify the split point:
   - Keep: frontmatter + heading + ## Goal + ## Quick reference (create one if it
     doesn't exist; use the most important 5–8 items from the skill)
   - Move: everything below that, starting from the first detailed section
3. Write the slimmed `.agents/skills/<name>/SKILL.md` (≤ 80 lines, ending with
   the `## Full guidance` pointer)
4. Write `.agents/skills/<name>/SKILL.deep.md` with the deep content

**Important for ## Quick reference**: if the skill doesn't currently have a
section with that name, look for an equivalent (e.g., "## Key commands",
"## Checklist", "## Common patterns") and use that content. If none exists,
create a minimal table of the 5 most important commands/concepts for the skill.

### STEP-BY-STEP for validate.sh

After splitting all skills, update `scripts/validate.sh` to add a check that:
- If a SKILL.md file references `SKILL.deep.md`, verify that SKILL.deep.md exists.

Find the existing skill validation block (around line 345-380). Add after the
version check block:

```bash
echo ""
echo "> Skill: SKILL.deep.md exists when referenced"
if [[ -d "$TARGET/skills" ]] && compgen -G "$TARGET/skills/*/SKILL.md" > /dev/null; then
    deep_missing=false
    for f in "$TARGET"/skills/*/SKILL.md; do
        [[ -f "$f" ]] || continue
        if grep -qF "SKILL.deep.md" "$f"; then
            deep_file="${f%SKILL.md}SKILL.deep.md"
            if [[ ! -f "$deep_file" ]]; then
                warn "${f#"$TARGET/"} references SKILL.deep.md but file is missing"
                deep_missing=true
            fi
        fi
    done
    $deep_missing || ok "all SKILL.deep.md references resolved"
else
    ok "no shared skills/ directory to check"
fi
```

Also apply this same block to `scripts/validate.ps1` (find the analogous
PowerShell block and add it there too, same pattern in PS syntax).
