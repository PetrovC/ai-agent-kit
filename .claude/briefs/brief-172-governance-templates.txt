### Background

Target projects that install the kit via `install.sh` get `docs/ai/` template files
from `project-template/` (PROJECT.md, ARCHITECTURE.md, COMMANDS.md, etc.).
The governance policy files (CONTEXT_GOVERNANCE, SUBAGENT_GOVERNANCE, MCP_POLICY,
MODEL_ROUTING, CONTEXT_SANITIZATION) currently only exist in the source repo's
`docs/ai/` and are NOT installed into target projects.

This task adds generic template versions of these 5 files to `project-template/`
so they get installed automatically on first install (skip-if-exists behavior is
already handled by install.sh — no script changes needed).

---

### Critical reading before you start

Read these files to understand their current content (they're in the source repo's
`docs/ai/` directory):
- `docs/ai/CONTEXT_GOVERNANCE.md`
- `docs/ai/SUBAGENT_GOVERNANCE.md`
- `docs/ai/MCP_POLICY.md`
- `docs/ai/MODEL_ROUTING.md`
- `docs/ai/CONTEXT_SANITIZATION.md`

Then read one existing project-template file (e.g. `project-template/ARCHITECTURE.md`)
to understand the template style — STOP notices, fillable sections, etc.

### Files to create

Create 5 files in `project-template/`. Each file is a GENERIC template version
of the corresponding governance doc. The rules:

**Keep** (generic, applies to any project):
- General policy principles and rules
- Table structures with placeholder values
- Conceptual guidance (e.g. "prefer read-only before write")

**Replace with STOP markers** (project-specific content the user must fill in):
- Specific AI model names (e.g. "claude-opus-4-8", "gpt-5.5")
- Specific subagent names and their assignments
- Specific reasoning effort values
- Provider-specific tool names that are kit-specific
- References to internal tooling files (e.g. "tooling/claude/CLAUDE.md")

**STOP notice format** (use exactly this for sections needing customization):
```
> ⚠️ **STOP** — Fill in your project-specific model assignments here.
```
