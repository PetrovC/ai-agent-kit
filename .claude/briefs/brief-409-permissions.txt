## Issue #409 — ci(scorecard): harden GitHub token permissions across workflows

**Task type:** ci
**Branch:** agent/agy/claude-sonnet-4-6/ci/token-permissions-409

---

### Start here

```bash
git checkout master
git pull origin master
git checkout -b agent/agy/claude-sonnet-4-6/ci/token-permissions-409
```

---

### Goal

Ensure every workflow has explicit, minimal top-level `permissions:` blocks.
Most workflows already have `permissions: contents: read`. The only workflow
that needs hardening is `agent-on-mention.yml`.

---

### STEP 1 — Harden `agent-on-mention.yml`

Open `.github/workflows/agent-on-mention.yml`.

Current top-level permissions:
```yaml
permissions:
  contents: write
  pull-requests: write
  issues: write
  id-token: write
```

The `id-token: write` permission grants OIDC token generation. The workflow uses
`CLAUDE_CODE_OAUTH_TOKEN` (an OAuth secret), NOT OIDC. Remove `id-token: write`.

Move the remaining permissions to the `claude` job level (not the top level),
so the top level is read-only:

Replace the current top-level `permissions:` block with:
```yaml
permissions:
  contents: read
```

And add a `permissions:` block inside the `claude` job:
```yaml
    permissions:
      contents: write      # create branches, push commits
      pull-requests: write # open / update PRs
      issues: write        # post comments
```

The resulting file should have:
- Top level: `permissions: contents: read`
- Job `claude`: explicit write permissions for the three scopes above

Do NOT change any other part of the workflow.

---

### STEP 2 — Verify `scorecard.yml`

Open `.github/workflows/scorecard.yml`. It currently has `permissions: read-all`
at the top level. This is the standard recommendation for Scorecard analysis
(the action needs broad read access). Leave it as is — no change needed.

---

### STEP 3 — Verify `release-checksums.yml`

Open `.github/workflows/release-checksums.yml`. It has `contents: write` and
`id-token: write` at the top level — these are needed for creating release
artifacts and signing them with Sigstore. Leave them as is — no change needed.

---

### STEP 4 — Update CHANGELOG.md

Under `## [Unreleased]` → `### Changed`, prepend:

```
  Removed unnecessary id-token:write. Moved write permissions to job level.

  Token-Permissions check.
```
