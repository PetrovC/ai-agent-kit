## Task: feat(claude): tune skillListingBudgetFraction + maxSkillDescriptionChars for 30 skills

### Context

This repository is a versioned configuration kit for Claude Code, Codex CLI, and Antigravity (agy).
It ships 30 shared skills. Claude Code exposes two settings to control how much context budget
skills consume: `skillListingBudgetFraction` (default 0.01) and `maxSkillDescriptionChars`
(default 1536).

With 30 skills each at 1536 chars, the listing can exceed useful size. The goal is to cap
description length at 600 chars so the listing stays readable without rewriting any skill prose.

### Files to modify

1. `tooling/claude/settings.json`   — add two keys at the top level (alongside "model", "cleanupPeriodDays", etc.)
2. `tooling/claude/settings.windows.json` — identical change (these two files share the same key set)
3. `docs/ai/MODEL_ROUTING.md` — add a brief "Skill listing budget" subsection under "Command and Agent Guidance"
4. `CHANGELOG.md` — add a feat(claude) entry under [Unreleased] > Added

### JSON keys to add

In both `settings.json` and `settings.windows.json`, at the top level (same level as "model"):

```json
"skillListingBudgetFraction": 0.01,
"maxSkillDescriptionChars": 600
```

### Documentation to add (MODEL_ROUTING.md)

Add a short subsection "Skill listing budget" after the "Command and Agent Guidance" section explaining:
- The kit ships 30 skills; Claude Code defaults list all skill descriptions (1536 chars each).
- `skillListingBudgetFraction: 0.01` — keep the default; this caps listing to 1% of context.
- `maxSkillDescriptionChars: 600` — cap descriptions at 600 chars to reduce per-skill token cost.
- Combined, 30 skills at 600 chars ≈ 18 k tokens at the cap vs 46 k at the default 1536 chars.
- Skill descriptions remain full-text in the actual SKILL.md files; only the listing summary is capped.
