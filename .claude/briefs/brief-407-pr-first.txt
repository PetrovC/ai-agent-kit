## Issue #407 — ci(scorecard): align workflows on PR-first execution policy

**Task type:** ci
**Branch:** agent/agy/claude-sonnet-4-6/ci/pr-first-407

---

### Start here

```bash
git checkout master
git pull origin master
git checkout -b agent/agy/claude-sonnet-4-6/ci/pr-first-407
```

---

### Goal

Make the trigger policy of every workflow explicit and appropriate. The only
workflow with a non-PR, non-schedule trigger that needs review is `powershell.yml`,
which has both `push: branches: [master]` AND `pull_request`.

---

### STEP 1 — Review `powershell.yml`

Open `.github/workflows/powershell.yml` and look at its `on:` block.
It currently runs on both `push: branches: [master]` AND `pull_request`.

This workflow runs PSScriptAnalyzer and uploads its SARIF output via
`github/codeql-action/upload-sarif`. For the GitHub Security tab to reflect
current status after each merge, the SARIF should be uploaded on push to master
too — not just on PRs.

Change: Keep both triggers but add a clear comment explaining why `push: master`
is retained. Edit the `on:` block to look like this (exactly):

```yaml
on:
  push:
    branches: ["master"]
    # SARIF upload needs push-to-master so the Security tab reflects the
    # current default-branch posture after each merge, not just during PRs.
  pull_request:
```

Do NOT remove `push: branches: [master]` from this file.

---

### STEP 2 — Classify all other workflows in a comment block

At the top of each workflow file (after the `name:` line, before `on:`), if a
classification comment is missing, add a short one-line comment explaining the
trigger rationale. Use this format:

```yaml
# Trigger: PR-blocking validation — runs on every pull request.
```

or

```yaml
# Trigger: Scheduled security scan — runs weekly + on branch-protection changes.
```

or

```yaml
# Trigger: Release event — runs when a GitHub release is published.
```

or

```yaml
# Trigger: Maintainer-invoked agent — runs when a maintainer comments @claude.
```

Classification:
- `pr-*.yml` files → "PR-blocking validation"
- `quality-gate.yml` → "PR-blocking validation"
- `powershell.yml` → "PR validation + SARIF push"
- `scorecard.yml` → "Scheduled security scan"
- `release-checksums.yml` → "Release event"
- `agent-on-mention.yml` → "Maintainer-invoked agent"
- `agent-codex-agy-on-mention.yml` → "Maintainer-invoked agent"

Only add a comment if one is not already there. Do not rewrite existing comments.
