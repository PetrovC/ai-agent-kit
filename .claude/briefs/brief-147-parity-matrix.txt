TASK: Add a CI workflow that enforces Bash and PowerShell script parity.

GOAL: Detect drift between Bash and PowerShell lifecycle scripts. ADR-001 flags this as a known risk.

BRANCH: Create branch agent/agy/claude-sonnet-4-6/ci/bash-ps-parity from master before making changes.

FIRST: Read the existing file .github/workflows/pr-install-parity.yml to understand what is already
covered, and avoid duplicating it.

WHAT TO CREATE:

File .github/workflows/pr-parity.yml — a GitHub Actions workflow.

Trigger: pull_request event, with paths filter covering:
  scripts/install.sh, scripts/install.ps1,
  scripts/update.sh, scripts/update.ps1,
  scripts/uninstall.sh, scripts/uninstall.ps1,
  scripts/validate.sh, scripts/validate.ps1

The workflow must have these jobs:

Job 1 — parity-bash (ubuntu-latest):
  - Checkout
  - Run bash scripts/install.sh with a temp target directory under the runner temp path
  - List all installed files (relative to target) and sort them
  - Save the sorted list as a file
  - Upload it as an artifact named bash-manifest

Job 2 — parity-powershell (windows-latest):
  - Checkout
  - Run pwsh scripts/install.ps1 with a temp target directory under the runner temp path
  - List all installed files (relative to target) and sort them
  - Save the sorted list as a file
  - Upload it as an artifact named ps-manifest

Job 3 — parity-diff (ubuntu-latest, needs both above jobs):
  - Checkout
  - Download both artifacts
  - Normalize path separators (replace backslashes with forward slashes)
  - Diff the two sorted file lists
  - If the diff is non-empty, print the differences and fail
  - Include a comment documenting how to add a legitimate exception if needed

Job name for the parity-diff job: bash-ps-parity

Do NOT add bash-ps-parity to .github/required-checks.txt — this is path-filtered.

Use actions/checkout at version 4 and actions/upload-artifact / download-artifact at version 4.

VERIFICATION: Validate the YAML with: python3 -c "import yaml; yaml.safe_load(open('.github/workflows/pr-parity.yml'))"
