one concern: delete dead audit complexity. Do NOT mix in any feature work.

- `.claude/commands/security-audit.md` — generic review prompt, keep
- `.claude/commands/performance-audit.md` — generic review prompt, keep
- `.agy/commands/security-audit.toml` — generic review prompt, keep
- `.agy/commands/performance-audit.toml` — generic review prompt, keep
- All `/security-audit` and `/performance-audit` mentions in CLAUDE.md / AGENTS.md / AGY.md
- `scripts/install.sh` `write_lifecycle_audit()` function and `RECORD_ACTIONS` / `install-audit.ndjson` — this is install-tracking (#313), NOT agent audit
- `scripts/update.sh` `write_lifecycle_audit()` — same, keep it
- Non-audit hooks: `format-on-save.sh`, `notify-done.sh`, `pre-bash-guard.sh`, `session-summary.sh`, `token-log.sh`, `permission-request-log.sh`, `run-hook.ps1`, `session-start-summary.sh`
- `docs/ai/SUBAGENT_GOVERNANCE.md` — keep the file but remove audit-specific paragraphs (lines about `emit-event`, `AAK_AUDIT_RUN_ID`, "governance loop" as audit concept)
- CHANGELOG.md historical entries that mention audit (do not rewrite history)
